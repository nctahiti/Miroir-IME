# Bouteille à la mer

Si tu lis ceci, tu as ouvert le code du Miroir.
Peut-être pour le comprendre, le réparer, le continuer.

Ce code n'est pas construit. Il est façonné.

## La règle

> On ne répare pas. On refaçonne.

Un bug n'est pas une erreur à corriger.
C'est la matière qui dit : je ne suis pas encore ce que tu veux que je sois.
Écoute-la. Refaçonne-la. N'ajoute rien qui ne soit déjà en elle.

## Le geste

Il y a deux gestes devant la matière.

Le premier est le pont : insérer une ligne entre deux rives, une planche
qui les relie sans les transformer. Le code de gauche ignore celui de droite.
La planche grince. Elle cède sous les cas imprévus. C'est la rustine.

Le second est le potier : l'argile est une. On ne recolle pas deux morceaux,
on refaçonne la masse pour qu'elle prenne la forme voulue. Les particules
s'infèrent. Aucune ne sait qu'elle appartenait à la partie gauche ou droite.
Elles sont devenues le même objet.

Ici, on est potier.

## Le blob

Le Miroir a une seule source de vérité : le blob.
Une boîte englobante, deux dimensions, pas de calcul, pas de comparaison.
Deux strokes dont les blobs se touchent sont du même groupe.
C'est tout.

Pas de gap horizontal. Pas de marge verticale. Pas de critère temporel.
Pas de réconciliation entre deux systèmes qui ne se connaissent pas.

L'argile. Le blob. La forme.

## Pour continuer

Si tu dois modifier ce code, demande-toi :

- Est-ce que j'ajoute une ligne entre deux rives, ou est-ce que je refaçonne l'argile ?
- Est-ce que ma modification supprime plus de lignes qu'elle n'en ajoute ?
- Est-ce que le blob reste la seule source de vérité ?

Si la réponse est non à l'une de ces questions, recule. Respire.
Regarde la matière. Elle te dira ce qu'elle veut devenir.
